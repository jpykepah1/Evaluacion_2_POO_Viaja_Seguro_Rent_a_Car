## 2025-02-14 - [Sanitización de Texto para Datos de Vehículos]
**Vulnerability:** Input for fields like `marca` and `modelo` in `gestion_vehiculos()` were being accepted directly from user input without validation or sanitization, potentially allowing arbitrary inputs including script injections (e.g., cross-site scripting risks or data format breaks when rendering) or empty inputs.
**Learning:** `validador_formatos.py` already includes a `sanitizar_texto` function designed exactly to mitigate these issues by removing dangerous characters, but it wasn't consistently applied to all raw inputs during object creation or updates. Relying solely on downstream SQL defenses ignores potential data corruption or UI injection.
**Prevention:** Always apply basic validation (e.g. non-empty check) and text sanitization (using `sanitizar_texto` or similar utility) directly at the user input point in the controller layer before the data propagates further down into DTOs or DAOs.
